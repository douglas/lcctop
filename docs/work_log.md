## Feature: Fix Ghostty tab focus for multi-window instances (TTY-based correlation)
- **Status:** closed
- **Started:** 2026-03-15T04:00:00-03:00
- **Closed:** 2026-03-15T05:30:00-03:00
- **Description:** Both lcctop-pick (Ruby) and Tauri picker fail to focus the correct Ghostty window when multiple windows share one PID. Fix uses TTY-based correlation: session.terminal.tty matched against tab-root tty_nr decoded from /proc, sorted by stableId/starttime for 1:1 window mapping.

## Feature: Fix Ghostty window focus — stableId hex parse + tab_root_pid correlation
- **Status:** closed
- **Started:** 2026-03-15T05:35:00-03:00
- **Closed:** 2026-03-15T06:00:00-03:00
- **Description:** Two bugs in the multi-window ghostty fix: (1) stableId is a hex string but parsed as decimal (.to_i truncates at first hex letter; Rust as_u64 returns None for strings) making the stableId sort completely wrong; (2) sessions with empty terminal:{} have no tty field so TTY matching falls back to first address. Fix: switch to tab_root_pid-based correlation (live process tree, no stored TTY needed) + fix stableId hex parsing.
